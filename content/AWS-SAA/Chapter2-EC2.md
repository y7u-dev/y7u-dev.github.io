# Chapter 2. EC2

## 인스턴스 타입

```
m5.xlarge
│ │  └── 크기 (nano < micro < small < medium < large < xlarge)
│ └──── 세대 (숫자 높을수록 최신)
└────── 패밀리 (용도)
```

| 패밀리 | 특화 | 용도 |
| --- | --- | --- |
| m | 균형잡힌 CPU/메모리 | 웹 서버, 일반 앱 |
| c | CPU 집약적 | 게임 서버, 인코딩 |
| r | 메모리 집약적 | DB, 캐시, 빅데이터 |
| i | 고속 로컬 스토리지 | NoSQL DB |
| p/g | GPU | 딥러닝, 렌더링 |

## 구매 옵션

| 옵션 | 특징 | 용도 |
| --- | --- | --- |
| 온디맨드 | 시간당 요금, 약정 없음 | 예측 불가능한 워크로드 |
| 예약 인스턴스 | 1/3년 약정, 최대 72% 할인 | 24/7 운영 서버 |
| 스팟 인스턴스 | 최대 90% 할인, AWS가 회수 가능 | 배치, ML 학습 |
| Savings Plans | 예약과 유사, 인스턴스 타입 유연 | 다양한 인스턴스 혼용 |

**설계 패턴:**
- 베이스 트래픽 → 예약 인스턴스
- 트래픽 급증 → 온디맨드 or 스팟
- 배치 처리 → 스팟 (중단/재시작 가능하게 설계)

## AMI (Amazon Machine Image)

OS + 소프트웨어 + 설정을 이미지로 저장. 동일한 서버 복제, 다른 리전 배포 가능.

## EBS vs 인스턴스 스토어

| 구분 | EBS | 인스턴스 스토어 |
| --- | --- | --- |
| 연결 | 네트워크 드라이브 | 물리적 부착 |
| 데이터 유지 | EC2 꺼져도 유지 | EC2 꺼지면 삭제 |
| 용도 | 일반 스토리지 | 캐시, 임시 파일 |

### EBS 볼륨 타입

| 타입 | 특징 | 용도 |
| --- | --- | --- |
| gp3 (SSD) | 범용, 비용 효율 | 일반 서버, 부팅 볼륨 |
| io2 (SSD) | 고IOPS | 고성능 DB |
| st1 (HDD) | 대용량, 순차 읽기 | 로그, 빅데이터 |
| sc1 (HDD) | 저비용 | 자주 안 쓰는 데이터 |

> **IOPS** = 디스크가 1초에 처리할 수 있는 읽기/쓰기 횟수
> 

## EC2 배치 그룹 (Placement Group)

| 종류 | 특징 | 용도 |
| --- | --- | --- |
| 클러스터 | 같은 AZ, 물리적으로 가깝게 | HPC, 초저지연 노드 간 통신 |
| 분산 | 서로 다른 물리 하드웨어 | 소규모 중요 인스턴스 동시 장애 방지 |
| 파티션 | 여러 파티션으로 분산 | Hadoop, Kafka 대규모 분산 시스템 |

## Auto Scaling

### 조정 방식

| 방식 | 기준 | 적합한 상황 |
| --- | --- | --- |
| 동적 조정 | CPU, SQS 큐 크기 등 | 예측 불가능한 트래픽 |
| 예약 조정 | 시간 스케줄 | 예측 가능한 패턴 |
| 예측 조정 | ML로 과거 패턴 분석 | 트래픽 급증 선제 대응 |
