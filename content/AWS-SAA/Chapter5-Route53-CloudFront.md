## Route 53 라우팅 정책

| 정책 | 동작 | 용도 |
| --- | --- | --- |
| 단순 | 항상 같은 IP | 서버 하나 |
| 가중치 기반 | 비율로 분산 | 카나리 배포 |
| 지연 시간 기반 | 가장 빠른 리전 | 글로벌 서비스 |
| 장애 조치 | 장애 시 백업 자동 전환 | 고가용성 |
| 지리적 | 국가/대륙 기준 | 규정 준수, 다국어 |

## CloudFront 동작 원리

```
최초 요청: 사용자 → 엣지 로케이션 → 오리진 → 캐시 저장
이후 요청: 사용자 → 엣지 로케이션 → 캐시에서 즉시 반환
```

전 세계 450개+ 엣지 로케이션에 콘텐츠 캐싱.

## 오리진 다중 설정

```
CloudFront
├── /static/* → S3 (정적, 캐싱 O)
└── /*        → ALB (동적, 캐싱 X)
```

하나의 CloudFront로 정적 + 동적 동시 처리 가능.

## CloudFront vs Global Accelerator

| 구분 | CloudFront | Global Accelerator |
| --- | --- | --- |
| 캐싱 | 있음 | 없음 |
| 주 용도 | 웹 콘텐츠 성능 개선 | TCP/UDP 레벨 라우팅 |
| 적합 상황 | 정적/동적 콘텐츠 | 게임, IoT, 실시간 앱 |

## ACM + CloudFront 중요 규칙

> CloudFront에 ACM 인증서를 사용하려면 반드시 **us-east-1 리전**에서 발급해야 함
ALB용 인증서는 ALB와 같은 리전에서 발급하면 됨
