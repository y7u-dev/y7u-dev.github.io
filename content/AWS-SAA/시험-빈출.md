# 추가 학습 (시험 빈출 서비스)

## 🔴 WAF (Web Application Firewall) - 74회 등장

ALB, CloudFront, API Gateway 앞에 붙이는 웹 방화벽.

**막을 수 있는 것:**
- SQL 인젝션, XSS
- 특정 IP/국가 차단
- 요청 속도 제한 (DDoS 방어)

**Shield와 차이:**

| 구분 | WAF | Shield |
| --- | --- | --- |
| 계층 | L7 (애플리케이션) | L3/L4 (네트워크) |
| 설정 | 규칙 직접 설정 | 자동 방어 |
| 비용 | 유료 | Standard 무료 / Advanced 유료 |

## 🔴 ECS + Fargate - 61 + 30회 등장

**ECS (Elastic Container Service):** 도커 컨테이너 실행/관리.

| 실행 방식 | 특징 |
| --- | --- |
| EC2 모드 | EC2 위에서 컨테이너 실행, 서버 직접 관리 |
| Fargate 모드 | 서버 없이 컨테이너만 실행 (서버리스 컨테이너) |

**Lambda vs Fargate:**

| 구분 | Lambda | Fargate |
| --- | --- | --- |
| 실행 단위 | 함수 | 컨테이너 |
| 시간 제한 | 최대 15분 | 없음 |
| 제어 수준 | 낮음 | 높음 |

**EKS:** 쿠버네티스를 AWS에서 관리형으로 제공. ECS보다 복잡하지만 K8s 기능 그대로 사용.

## 🔴 FSx - 48회 등장

| 서비스 | 특징 | 용도 |
| --- | --- | --- |
| FSx for Windows | SMB, Active Directory 통합 | Windows 환경 파일 공유 |
| FSx for Lustre | 초고속 병렬 파일 시스템, S3 통합 | HPC, ML 학습 |

**선택 기준:**

- **Windows 환경:** `FSx for Windows` (SMB/AD 필수)
- **초고속 연산 (ML/HPC):** `FSx for Lustre` (S3 연동 강점)
- **일반 Linux 공유:** `EFS` (관리 편의성 최고)
- **고성능 Linux 공유:** `FSx for OpenZFS` (속도와 지연 시간 중심)
- **복합 환경 (Linux+Win):** `FSx for NetApp ONTAP` (만능 스토리지)

## 🟡 Cognito - 26회 등장

앱 사용자 인증/인가 서비스.

| 구성 | 역할 |
| --- | --- |
| User Pool | 회원가입/로그인, 소셜 로그인, JWT 발급 |
| Identity Pool | 로그인한 사용자에게 AWS 리소스 접근 권한 부여 |

**IAM과 차이:**
- IAM: AWS 내부 사용자/서비스 권한
- Cognito: 외부 앱 사용자 인증 + AWS 접근

## 🟡 Step Functions - 15회 등장

여러 Lambda 함수나 AWS 서비스를 순서대로 조율하는 워크플로우.

```
[주문 접수] → [결제 처리] → 성공 → [재고 차감] → [알림]
                          → 실패 → [환불 처리]
```

**Lambda 체이닝과 차이:**
- Lambda 직접 호출: 에러/재시도 로직 직접 코딩
- Step Functions: 시각적 워크플로우, 에러/재시도 자동 관리

## 🟡 CloudFormation / Elastic Beanstalk - 22회 등장

**CloudFormation:** 인프라를 YAML/JSON 코드로 정의 (IaC). 모든 AWS 리소스 관리 가능.

**Elastic Beanstalk:** 앱 배포에 특화된 PaaS. 인프라 자동 구성.

| 구분 | CloudFormation | Elastic Beanstalk |
| --- | --- | --- |
| 제어 | 세밀한 제어 가능 | 자동 구성 |
| 복잡도 | 높음 | 낮음 |
| 용도 | 모든 인프라 코드 관리 | 빠른 앱 배포 |

## 🟡 Lambda Provisioned Concurrency

콜드 스타트 방지. 컨테이너를 미리 띄워두고 항상 웜 상태 유지.

**패턴:**

```
예약 조정 + Provisioned Concurrency:
오전 8:50 → Concurrency 늘림 (업무 시작 전)
오후 18:00 → Concurrency 줄임 (업무 종료 후)
→ 비용 최소화 + 콜드 스타트 방지
```
