### 1. 리눅스 소개

**탄생 배경**
1991년 리누스 토르발즈(당시 21세 대학생)가 유닉스에서 영감을 받아 개발. 상용 OS였던 유닉스를 개인용으로 만들고자 시작했고, 리처드 스톨먼의 GNU 프로젝트(컴파일러, 디버거 등 도구 모음)와 결합되어 현재의 GNU/Linux가 탄생.

**왜 리눅스를 쓰나?**
서버 운용 목적에서 오픈소스, 풍부한 레퍼런스, 보안 우수성이 핵심 이유. 엔드유저가 아닌 엔지니어 관점에서 접근.

**주요 배포판**

| 계열 | 배포판 | 특징 |
| --- | --- | --- |
| 데비안 | Ubuntu, Debian | 안정성, 광범위한 소프트웨어 지원 |
| 레드햇 | RHEL, CentOS, Fedora | 기업 환경 중심 |
| 아치 | Arch, Manjaro | 최신 소프트웨어, 고급 사용자용 |

수업에서는 **Ubuntu 22.04 LTS** 사용.

**GUI vs CLI**
서버는 원격으로 CLI를 통해 관리하는 것이 일반적. GUI는 불필요한 자원 소모를 유발하기 때문에 서버 환경에서는 CLI 모드로 운영.

```bash
# CLI 모드로 전환
sudo systemctl set-default multi-user.target
sudo reboot
```

**SSH 접속 (MobaXterm)**

```bash
# VM에 SSH 서버 설치
sudo apt update
sudo apt install openssh-server
sudo systemctl enable ssh

# MobaXterm 설정
# Remote host: 127.0.0.1 / Port: 2222 (포트 포워딩)
```

### 2. 파일 시스템

**파일 시스템이란?**
저장 장치에 데이터를 체계적으로 저장하고 관리하는 방법과 규칙. 저장장치 자체는 0과 1의 집합일 뿐이므로, 파일 시스템이 어디에 어떻게 저장할지 규칙을 제공.

**주요 디렉토리 구조**

| 경로 | 역할 |
| --- | --- |
| `/` | 루트 — 모든 파일/디렉토리의 시작점 |
| `/bin` | 기본 명령어 실행 파일 (ls, cp, mv 등) |
| `/sbin` | 시스템 관리자용 명령어 (root 권한 필요) |
| `/etc` | 시스템 및 애플리케이션 설정 파일 |
| `/home` | 일반 사용자 홈 디렉토리 |
| `/root` | root 사용자 홈 디렉토리 |
| `/var` | 가변 데이터 (로그, 캐시 등) `/var/log` |
| `/tmp` | 임시 파일 (재부팅 시 삭제) |
| `/usr` | 사용자 설치 프로그램 및 라이브러리 |
| `/dev` | 하드웨어 디바이스 파일 |
| `/lib` | 핵심 라이브러리 파일 |
| `/opt` | 선택적 애플리케이션 설치 경로 |

### 3. 리눅스 명령어

**명령어 기본 구조**

```
명령어 [옵션] [인자]
# 옵션: -로 시작 (단축), --로 시작 (전체)
# 예: ls -la /home
```

**명령어 실행 원리**
사용자 입력 → 쉘(bash)이 해석 → PATH에서 실행 파일 탐색 → 실행 → 결과 출력. 명령어는 결국 `/bin` 등에 있는 실행 파일을 호출하는 수단.

**파일/디렉토리 관리**

| 명령어 | 역할 | 주요 옵션 |
| --- | --- | --- |
| `ls` | 목록 조회 | `-l` 상세, `-a` 숨김파일, `-h` 용량표시, `-R` 하위포함 |
| `cd` | 디렉토리 이동 | `..` 상위, `~` 홈 |
| `mkdir` | 디렉토리 생성 | `-p` 중간 경로 자동 생성 |
| `rm` | 삭제 | `-r` 디렉토리, `-f` 강제 |
| `cp` | 복사 | `-r` 디렉토리, `-i` 덮어쓰기 확인 |
| `mv` | 이동/이름변경 | `-i` 덮어쓰기 확인, `-f` 강제 |
| `touch` | 빈 파일 생성 |  |
| `find` | 파일 검색 | `-name`, `-type` |

**파일 내용 보기/편집**

| 명령어 | 역할 |
| --- | --- |
| `cat` | 파일 내용 출력 (`-n` 줄번호) |
| `more` | 페이지 단위 출력 (Space: 다음, q: 종료) |
| `less` | more보다 향상된 버전 (상하 스크롤) |
| `head` | 파일 앞부분 출력 (기본 10줄) |
| `tail` | 파일 뒷부분 출력 (기본 10줄) |
| `grep` | 패턴 검색 (`grep "ERROR" server.log`) |
| `nano` | 간단한 텍스트 편집기 |
| `vim` | 강력한 텍스트 편집기 |

**vim 주요 단축키**

```
i        → 입력 모드
Esc      → 명령 모드
:w       → 저장
:q       → 종료
:wq      → 저장 후 종료
:q!      → 강제 종료 (저장 안 함)
```

**시스템 정보 확인**

| 명령어 | 역할 |
| --- | --- |
| `uname -a` | 전체 시스템 정보 |
| `df -h` | 디스크 공간 사용량 |
| `du -sh [경로]` | 특정 디렉토리 용량 |
| `top` | 실시간 프로세스/자원 모니터링 |
| `ps` | 현재 프로세스 목록 |
| `free` | 메모리 사용량 |
| `pwd` | 현재 경로 출력 |

**리디렉션과 파이프**

```bash
# 파이프 - 앞 명령어 출력을 다음 명령어 입력으로 전달
ls -l | grep ".txt"

# 리디렉션 - 출력을 파일로 저장 (덮어쓰기)
echo "Hello" > output.txt

# 리디렉션 - 출력을 파일에 추가 (append)
echo "World" >> output.txt

# 조합 활용
ps -ef | grep bash > bash_processes.txt
grep "ERROR" server.log | grep "pointer"
```

### 4. 사용자 및 권한 관리

**사용자 관리 명령어**

| 명령어 | 역할 |
| --- | --- |
| `useradd` | 사용자 추가 |
| `usermod` | 사용자 정보 수정 |
| `userdel` | 사용자 삭제 |
| `passwd` | 비밀번호 변경 |
| `groupadd/mod/del` | 그룹 관리 |
| `sudo` | 관리자 권한으로 명령 실행 |

**파일 권한 (chmod)**

```bash
# 권한 표기: rwxrwxrwx (소유자/그룹/기타)
# r=4, w=2, x=1

chmod 755 file.txt   # rwxr-xr-x
chmod 644 file.txt   # rw-r--r--
chown user:group file.txt   # 소유자 변경
```

### 5. 서비스 및 프로세스 관리

**systemctl 주요 명령어**

```bash
sudo systemctl start [서비스]      # 시작
sudo systemctl stop [서비스]       # 중지
sudo systemctl restart [서비스]    # 재시작
sudo systemctl enable [서비스]     # 부팅 시 자동 시작
sudo systemctl status [서비스]     # 상태 확인
```

**프로세스 관리**

```bash
ps -ef              # 전체 프로세스 목록
kill [PID]          # 프로세스 종료
kill -9 [PID]       # 강제 종료
```
