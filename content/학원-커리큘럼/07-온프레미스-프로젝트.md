### 프로젝트 내용

**Before → After 아키텍처**

| 항목 | Before (3-Tier) | After (Kubernetes) |
| --- | --- | --- |
| 처리 방식 | 동기 처리, DB 직결 | Redis Queue 비동기 처리 |
| 확장 방식 | 서버 전체 수동 확장 | Pod 단위 HPA 자동 확장 |
| DB 구성 | 단일 DB (SPOF) | PostgreSQL Primary + Replica |
| 배포 방식 | 수동 (10~30분) | ArgoCD GitOps 자동 (1~5분) |
| 모니터링 | 부재 | Prometheus + Grafana + Slack |
| 외부 접근 | 공인 IP 직접 노출 | Cloudflare Tunnel |

**인프라 구성 (단일 노트북 기반 VM 7대)**

| VM | 역할 | 사양 |
| --- | --- | --- |
| Bastion | 외부 접근 제어, SSH 점프 서버 | 1vCPU / 1GB |
| CP-1 (Main) | K8s Control Plane | 3vCPU / 5GB |
| CP-2 (HA) | K8s Control Plane (HA) | 2vCPU / 4GB |
| W-1 | Worker — Backend + Frontend Pod | 2vCPU / 4GB |
| W-2 | Worker — Tunnel + Grafana + ArgoCD | 2vCPU / 4GB |
| plask-db | PostgreSQL Primary + Replica | 2vCPU / 4GB |
| plask-redis | Redis Session + Queue | 2vCPU / 4GB |

**기술 스택**
Kubernetes · Calico CNI · MetalLB · Ingress-nginx · ArgoCD · Helm · Prometheus · Grafana · Alertmanager · PostgreSQL · Redis · Cloudflare Tunnel · cert-manager · k6

### 담당 역할

**개인 담당 — 네트워크 보안 설계 및 구현 (GNS3)**

| 영역 | 내용 |
| --- | --- |
| GNS3 토폴로지 설계 | VMnet4(K8s 10.127.111.0/24) / VMnet5(DB 172.16.40.0/24) 이중 서브넷 구조 |
| Router ACL (1차 방어) | K8s → DB 허용, 외부 → DB 전체 차단 (Whitelist 방식) |
| HSRP + Interface Tracking | Active/Standby 게이트웨이 이중화, Failover 약 1초 실측 검증 |
| Firewall VM iptables (2차 방어) | Stateful Inspection, NULL/XMAS/SYN-FIN 포트스캔 방어 |
| MASQUERADE | Pod CIDR(192.168.x.x) → Node IP NAT으로 비대칭 라우팅 해소 |
| cert-manager TLS | self-signed ClusterIssuer + Ingress TLS 설정 |

**공통 참여 — 팀 전체 설계 및 구축**

| 영역 | 내용 |
| --- | --- |
| 아키텍처 설계 | Before/After 아키텍처 분석, 네트워크 분리 설계, 전체 물리 구성도 작성 |
| Kubernetes 클러스터 | kubeadm 기반 HA 클러스터 구축 (CP 이중화, etcd 포함) |

| MetalLB BGP | 온프레미스 LoadBalancer IP 할당 구성 |
| Ingress-nginx | 경로 기반 라우팅, TLS 종료 처리 |
| Prometheus / Grafana | 모니터링 스택 구성, Alertmanager + Slack 알림 연동 |
| k6 부하 테스트 | Before/After 성능 비교 테스트 설계 및 수행 |

### 핵심 트러블슈팅

| # | 문제 | 원인 | 해결 |
| --- | --- | --- | --- |
| 1 | Worker 노드 인터넷 완전 차단 | Calico BIRD가 VMware NAT 게이트웨이 경로를 BGP로 덮어씀 | `IP_AUTODETECTION_METHOD=interface=ens34` 고정 |
| 2 | ping 시 DUP 패킷 루프 발생 | GNS3 클라우드 노드에 동일 VMnet 어댑터 중복 연결 | VMnet 어댑터 1:1 매핑 원칙 준수 |
| 3 | GNS3 Linked Clone 시작 불가 | VMX 파일 내 한글 경로가 CP949로 저장 → GNS3 UTF-8 파싱 실패 | VMX 파일 UTF-8 변환, 경로 ASCII-only로 변경 |
| 4 | K8s → DB 통신 전체 차단 | Router ACL에서 deny 규칙이 permit 규칙보다 상위에 배치됨 | permit 규칙을 deny 규칙보다 상위로 재배치 |
