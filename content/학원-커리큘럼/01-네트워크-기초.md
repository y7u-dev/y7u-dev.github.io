# 네트워크 기초

**학습 기간:** 2026.12.29 ~ 2026.01.13

---

### 1. 네트워크 기초 개념

**네트워크란?**
두 대 이상의 컴퓨터가 데이터를 주고받을 수 있도록 연결된 통신 시스템

**3가지 필수 요소**

- 물리적 연결 — 케이블 또는 무선 신호로 장치를 실제 연결
- 주소 체계 — 각 장치를 구분하는 고유한 주소 (IP)
- 프로토콜 — 데이터를 주고받는 규칙과 약속

**규모별 네트워크 분류**

| 종류 | 범위 | 소유권 |
| --- | --- | --- |
| PAN | 개인 주변 기기 | 개인 |
| LAN | 건물/구내 | 개인/회사 |
| MAN | 도시 단위 | 기관 |
| WAN | 국가/대륙 | KT, SKT 등 ISP |

### 2. IP 주소 / 서브네팅

**IP 주소 구조**

- 32비트를 8비트씩 4개 옥텟으로 나눠 10진수로 표현
- 예: `192.168.10.10` → `11000000.10101000.00001010.00001010`

**서브넷 마스크 / CIDR**

- 서브넷 마스크: 네트워크 주소와 호스트 주소를 구분하는 비트 마스크
- CIDR: `/24` 형식으로 서브넷 마스크를 간결하게 표현
- 예: `192.168.10.0/24` → 호스트 범위 `192.168.10.1 ~ 192.168.10.254`

**서브네팅이 필요한 이유**

- 하나의 큰 네트워크를 작은 단위로 나눠 브로드캐스트 도메인 축소 및 보안 강화

**게이트웨이 / DNS / DHCP**

| 개념 | 역할 |
| --- | --- |
| Default Gateway | 다른 네트워크로 나가는 출입구 (보통 라우터) |
| DNS | 도메인 이름 → IP 주소 변환 |
| DHCP | IP 주소를 동적으로 자동 할당 |

### 3. OSI 7계층 / TCP/IP 4계층

**OSI 7계층**

| 계층 | 이름 | 역할 | 주요 장비/프로토콜 |
| --- | --- | --- | --- |
| 7 | Application | 사용자 인터페이스 | HTTP, FTP, DNS |
| 6 | Presentation | 데이터 형식 변환 | SSL/TLS |
| 5 | Session | 세션 관리 |  |
| 4 | Transport | end-to-end 전송 | TCP, UDP |
| 3 | Network | 경로 결정 | IP, 라우터 |
| 2 | Data Link | 인접 장치 간 전송 | MAC, 스위치 |
| 1 | Physical | 물리적 신호 전달 | 케이블, NIC |

**TCP/IP 4계층** (실제 인터넷 표준)

| TCP/IP | OSI 대응 |
| --- | --- |
| Application | 5, 6, 7계층 |
| Transport | 4계층 |
| Internet | 3계층 |
| Network Access | 1, 2계층 |

**PDU (Protocol Data Unit)**

- L4: Segment / L3: Packet / L2: Frame / L1: Bit

**스위치 vs 라우터**

|  | 스위치 (L2) | 라우터 (L3) |
| --- | --- | --- |
| 주소 | MAC 주소 | IP 주소 |
| 범위 | 같은 네트워크 내 | 네트워크 간 |
| 역할 | 프레임 전달 | 경로 결정 |

### 4. VLAN

**개념**
물리적 구성과 무관하게 스위치를 논리적으로 분리하는 기술. 같은 스위치에 연결되어 있어도 다른 VLAN이면 브로드캐스트 도메인이 분리됨

**VLAN vs 서브넷**

- VLAN: L2 레벨의 논리적 분리
- 서브넷: L3 레벨의 IP 주소 분리
- 실제 운영에서는 VLAN 하나당 서브넷 하나를 매핑하는 방식으로 사용

**포트 종류**

| 포트 | 역할 |
| --- | --- |
| Access Port | 단일 VLAN에만 속함, 일반 단말 연결 |
| Trunk Port | 여러 VLAN 트래픽을 태그(802.1Q)로 구분해 전달, 스위치 간 연결 |

### 5. 라우팅

**라우팅 테이블 구성 요소**

- 목적지 네트워크: 어디로 가야 하나
- 인터페이스: 어느 포트로 내보내야 하나
- 넥스트 홉: 다음에 거칠 라우터 IP

**경로 종류**

| 종류 | 설명 |
| --- | --- |
| Connected | 직접 연결된 인터페이스 경로 |
| Static | 관리자가 수동으로 설정 |
| Default | 일치하는 경로가 없을 때 사용 (0.0.0.0/0) |
| Dynamic | 라우팅 프로토콜이 자동으로 학습 |

**정적 vs 동적 라우팅**

|  | 정적 | 동적 |
| --- | --- | --- |
| 설정 | 수동 | 자동 |
| 변화 대응 | 불가 (수동 수정 필요) | 자동 재계산 |
| 적합한 환경 | 소규모 고정 망 | 대규모 변동 망 |

**동적 라우팅 프로토콜**

| 프로토콜 | 분류 | 특징 |
| --- | --- | --- |
| RIP | IGP | 홉 수 기준, 최대 15홉, 소규모 적합 |
| OSPF | IGP | 링크 상태 기반, 대규모 기업 망 적합 |
| BGP | EGP | AS 간 라우팅, 인터넷 백본, ISP 간 사용 |

### 6. NAT

**개념**
사설 IP ↔ 공인 IP 주소를 변환하는 기술. 공인 IP 절약 + 내부 망 보안 효과

**종류**

| 종류 | 설명 |
| --- | --- |
| Static NAT | 사설 IP 1개 ↔ 공인 IP 1개 고정 매핑 |
| Dynamic NAT | 공인 IP 풀에서 동적 할당 |
| PAT (Overload) | 여러 사설 IP → 공인 IP 1개 (포트 번호로 구분), 가장 일반적 |

**와일드카드 마스크**

- ACL에서 사용하는 마스크. 서브넷 마스크와 반대 개념 (0=매칭, 1=무시)
- 예: `/24` → 와일드카드 `0.0.0.255`

### 7. ACL (Access Control List)

**개념**
라우터 인터페이스에서 패킷의 출발지/목적지 IP, 포트를 기준으로 트래픽을 허용/차단하는 필터링 정책

**동작 원리**

- 규칙을 위에서 아래로 순서대로 매칭
- 첫 번째 일치 규칙 즉시 적용, 이후 규칙 검사 안 함
- 마지막에 암묵적 `deny all` 존재

**종류**

| 종류 | 필터링 기준 | 적용 위치 |
| --- | --- | --- |
| Standard ACL | 출발지 IP만 | 목적지에 가깝게 |
| Extended ACL | 출발지/목적지 IP + 포트 | 출발지에 가깝게 |

**인터페이스 적용 방향**

- `inbound`: 들어오는 패킷에 적용
- `outbound`: 나가는 패킷에 적용
